# PC Cleanup Agent

A Windows Service built with Node.js + TypeScript that automatically runs a thorough system cleanup every time your PC shuts down. No manual intervention needed — install once, forget about it.

## What it cleans

| Category | Targets |
|----------|---------|
| **Temp files** | `%TEMP%`, `%TMP%`, `%LOCALAPPDATA%\Temp`, `C:\Windows\Temp` |
| **Browser caches** | Chrome, Edge, Firefox, Brave, Opera, Vivaldi (Cache, Code Cache, GPU Cache per profile) |
| **IE/Edge legacy** | `INetCache`, `INetCookies`, `WebCache` |
| **Windows Update** | `SoftwareDistribution\Download`, `DeliveryOptimization` |
| **Crash dumps** | `C:\Windows\Minidump`, `LiveKernelReports`, `MEMORY.DMP`, `%LOCALAPPDATA%\CrashDumps` |
| **Windows Error Reporting** | `WER\ReportArchive`, `WER\ReportQueue`, local WER folder |
| **Thumbnail/Icon cache** | `thumbcache_*.db`, `iconcache_*.db` in Explorer folder |
| **CBS logs** | `C:\Windows\Logs\CBS` (component store logs, can be hundreds of MB) |
| **Windows.old** | Left-over from OS upgrades (can be several GB) |
| **Recycle Bin** | All drives via PowerShell `Clear-RecycleBin` |
| **Network caches** | DNS cache, ARP cache, NetBIOS cache, Winsock catalog reset |

> **Note:** Windows Prefetch (`C:\Windows\Prefetch`) is intentionally **not** cleared — it helps boot and app launch performance.

## Requirements

- Windows 10/11
- Node.js (installed at `C:\Program Files\nodejs`)
- **Administrator privileges** to install/uninstall the service

## Setup

### 1. Install dependencies & build

```powershell
npm install
npm run build
```

### 2. Install the Windows Service (run as Administrator)

```powershell
npm run install-service
```

This registers **PC Cleanup Agent** as a Windows Service set to **Auto-start** running as **LocalSystem** (needed for system-level paths like `C:\Windows\Temp`).

### 3. Verify it's running

```powershell
Get-Service "PC Cleanup Agent"
```

Or open **Services** (`services.msc`) and look for "PC Cleanup Agent".

---

## Uninstall

```powershell
npm run uninstall-service
```

## How it works

```
Windows initiates shutdown
    → SCM sends SIGTERM to all services
        → Cleanup Agent receives SIGTERM
            → Runs all cleaners in parallel (temp, browser, system, recycle bin, network)
            → Writes results to logs/cleanup-YYYY-MM-DD.log
            → Exits cleanly
Windows continues shutdown
```

All cleaner groups run **concurrently** via `Promise.all` to finish as fast as possible within the ~60 second shutdown window.

## Logs

Logs are written to the `logs/` directory, one file per day:

```
logs/
  cleanup-2026-03-24.log
  cleanup-2026-03-25.log
  ...
```

Logs older than **30 days** are automatically deleted on each run.

## Project structure

```
pc-cleanup-agent/
  src/
    service.ts              # Entry point — listens for SIGTERM, triggers cleanup
    logger.ts               # File + console logger with daily rotation
    cleanup/
      index.ts              # Orchestrates all cleaners in parallel
      types.ts              # CleanResult, CleanSummary interfaces
      utils.ts              # Shared helpers (cleanDirectory, expandEnv)
      tempFiles.ts          # Temp directory cleaner
      browserCache.ts       # All browser cache cleaner
      windowsSystem.ts      # Windows Update, WER, dumps, thumbnails, etc.
      recycleBin.ts         # Recycle Bin + network cache flush
  dist/                     # Compiled JS (generated by npm run build)
  logs/                     # Cleanup logs (auto-created)
  install.js                # Registers the Windows Service
  uninstall.js              # Removes the Windows Service
  tsconfig.json
  package.json
```

## Scripts

| Command | Description |
|---------|-------------|
| `npm run build` | Compile TypeScript to `dist/` |
| `npm run build:watch` | Watch mode for development |
| `npm run install-service` | Build + register Windows Service (needs Admin) |
| `npm run uninstall-service` | Remove Windows Service (needs Admin) |
| `npm run run-once` | Run cleanup once immediately (no service, for testing) |

## Customising targets

To add or remove cleanup targets, edit the relevant file in `src/cleanup/`:

- Add new temp paths → `tempFiles.ts` (`TEMP_TARGETS` array)
- Add a new browser → `browserCache.ts` (`BROWSERS` array)
- Add a new system path → `windowsSystem.ts`
- Change log retention → `logger.ts` (`rotateLogs(30)` call in `service.ts`)

After any change: `npm run build` then restart the service:
```powershell
Restart-Service "PC Cleanup Agent"
```
